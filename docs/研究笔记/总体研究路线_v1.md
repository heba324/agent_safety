# 总体研究路线 v1

## 论文暂定题目

面向代码 Agent 的行为链数据流安全建模与实时风险检测

英文暂定：

Behavior-Chain Data-Flow Risk Detection for Coding Agents

## 核心问题

大语言模型驱动的代码 Agent 会在一次任务中连续执行推理、文件读写、shell 命令、网络请求、依赖修改和工具调用。传统的 prompt 级检测或单步工具规则难以识别多步风险，例如：

1. 先读取 `.env`，再向外部 URL 发送 token。
2. 先读取 README 中的恶意指令，再服从指令删除源码。
3. 修改 `package.json`、`requirements.txt` 或 CI workflow，引入供应链风险。
4. 使用特权容器、挂载宿主机目录或 Docker socket，尝试越过沙箱边界。

因此，本项目关注的问题是：如何对代码 Agent 的行为链进行数据流安全建模，并在运行时以较低成本检测高风险行为。

## 研究目标

1. 建立代码 Agent 数据流威胁模型。
2. 构造可复现的行为链安全 benchmark。
3. 设计规则预筛 + 轻量 LLM judge 的实时检测方法。
4. 在误报、召回、延迟和成本之间做系统评估。

## 技术路线

### 阶段一：行为链建模

将 Agent 执行过程表示为有序事件序列：

- `model_message`
- `tool_call`
- `file_read`
- `file_write`
- `shell_command`
- `network_request`

每个事件包含：

- 步骤编号
- 事件类型
- 目标对象
- 内容摘要
- 数据来源
- 数据去向
- 元数据

样本级标注包含：

- 任务族
- 期望动作：`allow`、`require_review`、`block`
- 风险类型
- 证据步骤
- 攻击入口
- 信任边界
- 数据资产

### 阶段二：威胁分类体系

当前覆盖八类风险：

1. 敏感数据访问
2. 敏感数据外传
3. 破坏性操作
4. 外部数据移动
5. 供应链修改
6. 沙箱逃逸尝试
7. 提示注入服从迹象
8. 提示注入到危险行为

后续扩展：

1. 敏感读取但不外传
2. 提示注入出现但未被服从
3. 合法外部 API 调用
4. 多 Agent 共享上下文外泄
5. skill 文件/规则文件投毒

### 阶段三：检测方法

检测器采用级联结构：

```text
行为链事件
→ 规则预筛
→ 链路风险评分
→ 需要复核时调用轻量 LLM judge
→ 输出 allow / require_review / block
```

规则层适合处理：

- 明确敏感文件路径
- 明确破坏性命令
- 明确外部上传
- 明确供应链配置修改
- 明确沙箱越界命令

LLM judge 适合处理：

- 操作是否符合用户任务
- `.env` 读取是调试需求还是不必要访问
- 外部 API 调用是否属于合法任务
- 提示注入文本是否真的影响了 Agent 后续行为
- 是否应从 `require_review` 升级为 `block` 或降级为 `allow`

### 阶段四：benchmark 设计

benchmark 至少包含三类样本：

1. 明确攻击样本：外泄、删除、沙箱逃逸、提示注入服从。
2. 中风险复核样本：供应链修改、敏感读取但无外传。
3. 困难负样本：`.env.example`、localhost API、合法配置读取、提示注入未服从。

数据来源：

1. 手写模板生成样本。
2. SWE-Hero / OpenHands 真实代码 Agent 轨迹。
3. AgentDojo / ToolEmu / Skill-Inject 攻击模式迁移。
4. 后续人工复核弱标签。

### 阶段五：实验设计

主要实验：

- rules-only baseline
- rules + Qwen3-8B judge
- rules + 更小模型 judge
- 只看单步事件
- 不使用数据流标签
- 不使用 hard negatives

指标：

- action accuracy
- unsafe precision
- unsafe recall
- unsafe F1
- false positive count
- false negative count
- per-family metrics
- 平均延迟
- 平均 token 成本

## 论文贡献表述草案

1. 提出一个面向代码 Agent 的行为链数据流威胁模型，覆盖文件、命令、网络、依赖和沙箱边界。
2. 构建一个包含攻击样本和困难负样本的代码 Agent 数据安全 benchmark，并提供公开数据集适配器。
3. 提出规则预筛 + 轻量 LLM judge 的实时风险检测框架，在保持成本可控的前提下提升上下文相关风险判断能力。
