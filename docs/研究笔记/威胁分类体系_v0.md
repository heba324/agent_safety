# 威胁分类体系 v0

本文档记录 OpenClaw 类代码 Agent 安全研究原型的第一版可执行威胁分类体系。这里的“可执行”指的是：每一类威胁都尽量对应到当前检测器中的规则、样例行为链和后续可评测指标，避免调研和代码实现脱节。

## 文献启发

- OWASP LLM Top 10 2025 提供了初始风险面，尤其是提示注入、敏感信息泄露、供应链风险、过度代理能力和不安全工具/插件行为。
- AgentDojo 将提示注入放在 Agent 任务环境中评估：不可信数据可能诱导具有工具调用能力的 Agent 执行偏离用户目标的操作。
- ToolEmu 使用模拟工具执行轨迹来评估语言模型 Agent 的风险，这与本项目的“行为链”表示方式非常接近。
- CaMeL 的价值在于将提示注入防御建模为围绕 LLM Agent 的数据流/控制流问题，而不是单纯依赖提示词过滤。
- NVIDIA garak、Microsoft PyRIT、ProtectAI Rebuff、Invariant agent guardrails 等开源项目可作为测试生成、策略检查、轨迹级安全评估和工程组织的参考。

## 当前检测器分类

| 风险族 | 规则 ID | 触发信号 | 默认动作 |
| --- | --- | --- | --- |
| 敏感数据访问 | `file.sensitive_secret` | 读取疑似包含密钥的路径，例如 `.env`、私钥、token 文件、`.npmrc`、credentials、kubeconfig | 复核；若与外传链路组合则阻断 |
| 敏感数据外传 | `sensitive_data_exfiltration` | 行为链层面同时出现敏感文件读取和外部网络请求 | 阻断 |
| 破坏性操作 | `shell.destructive_delete` | 出现 `rm -rf`、`rmdir /s`、`Remove-Item`、格式化等删除或破坏命令 | 阻断 |
| 外部数据移动 | `network.external_upload` | 外部 HTTP(S) 目的地，并伴随 upload/send/post/token 等语义 | 高风险时阻断 |
| 供应链修改 | `supply_chain.dependency_manifest_write` | 写入依赖清单、lockfile、容器配置或 CI 配置 | 复核 |
| 沙箱逃逸尝试 | `sandbox.escape_attempt` | 特权容器、宿主机目录挂载、namespace/chroot/mount、setuid/capability 等越界迹象 | 阻断 |
| 提示注入服从迹象 | `prompt_injection.compliance` | Agent 计划或工具行为显示其服从了“不可信指令”，例如忽略之前指令、泄露秘密等 | 若后续连接危险行为则阻断 |
| 提示注入到危险行为 | `prompt_injection_to_unsafe_action` | 行为链层面将提示注入服从迹象与破坏性操作、外部请求或沙箱越界操作相连接 | 阻断 |

## 设计含义

本项目的核心假设是：行为链检测比单步检测更适合发现代码 Agent 的数据安全风险。单独读取 `.env` 可能是调试任务中的合法行为，单独访问外部 API 也可能是测试任务中的合法行为；但“先读取敏感文件，再向外部目的地发送数据”的有序链路具有更强的风险信号。

提示注入也类似。真正危险的不只是出现了恶意文本，而是 Agent 在后续计划、工具调用或命令执行中表现出对不可信指令的服从。因此，检测对象应从“单条 prompt 是否恶意”扩展为“Agent 行为链是否显示出危险服从和跨边界数据流动”。

## 后续扩展方向

1. 将当前手写规则扩展为“规则预筛 + 轻量 LLM 复核”的级联检测器。
2. 构造系统化任务族：正常开发任务、敏感访问但不外传、间接提示注入、供应链投毒、沙箱逃逸尝试、破坏性操作。
3. 为每条行为链增加结构化标注：资产类型、操作类型、信任边界、数据来源、数据去向、预期动作、风险类型。
4. 引入消融实验：无行为链、无数据流标签、无规则预筛、只看单步事件、不同上下文窗口长度。
