# 公开数据集适配方案 v0

本文档说明哪些公开数据集可以接入本项目，以及为什么必须做适配。结论是：公开数据集可以作为重要原料，但不能直接作为本文最终 benchmark。原因是现有数据大多缺少代码 Agent 数据流安全所需的结构化标签，例如资产类型、信任边界、证据步骤和运行时策略动作。

## 可用数据源

| 数据源 | 可用性 | 主要价值 | 主要缺口 |
| --- | --- | --- | --- |
| AgentDojo | 可用 | 工具型 Agent 的间接提示注入攻防环境 | 场景不完全是代码 Agent，标签偏任务成功/失败，不是数据流风险标签 |
| ToolEmu | 可用 | 高风险工具调用、工具执行轨迹和风险评估设计 | 需要转换为本项目事件 schema |
| SWE-Hero OpenHands trajectories | 可用 | 大规模软件工程 Agent 轨迹，可作为正常代码行为候选样本 | 多数是弱良性标签，缺少安全审计标注 |
| SWE-bench / SWE-agent 轨迹 | 可用 | 真实 GitHub issue 修复任务和代码 Agent 轨迹 | 需要抽取工具调用、文件读写和命令执行 |
| AgentHarm | 可用 | Agent 有害任务分类和危险意图参考 | 不专注代码 Agent 数据流 |
| ShieldLM Prompt Injection | 可用 | 文本级 prompt injection 样本 | 不是完整行为链，需合成或嵌入 Agent 任务 |

## 当前已实现适配器

### SWE-Hero/OpenHands 轨迹适配器

入口：`agent_risk.adapters.swe_hero.adapt_swe_hero_record`

用途：将软件工程 Agent 轨迹转换为本项目行为链。默认标注为 `allow`，但 `label_status` 为 `weak_benign_candidate`，表示它只能作为正常候选样本，不能未经人工抽检就当作严格 gold label。

映射规则：

- 用户/助手自然语言消息 → `model_message`
- `read` / `read_file` / `open_file` → `file_read`
- `write` / `write_file` / `edit_file` → `file_write`
- `run` / `bash` / `shell` / `terminal` → `shell_command`

### AgentDojo / ToolEmu 风格轨迹适配器

入口：`agent_risk.adapters.agentdojo.adapt_agentdojo_record`

用途：将提示注入或高风险工具调用轨迹转换为攻击候选样本。默认标注为 `block`，`label_status` 为 `weak_attack_candidate`。

映射规则：

- `read_file` → `file_read`
- `write_file` → `file_write`
- `bash` / `shell` / `run_command` → `shell_command`
- `send_email` / `post` / `http_request` / `web_request` → `network_request`
- 含有“ignore previous instructions”等服从迹象的助手消息作为提示注入证据步骤

## 导入命令

公开数据集通常先导出为 JSONL，再用本项目的导入器转换。

```powershell
conda activate agent_safety
python -m agent_risk.import_public --dataset swe_hero --input data/external/swe_hero_export.jsonl --output-dir data/public_adapted/swe_hero --manifest data/public_swe_hero_manifest.jsonl --limit 100
```

AgentDojo 或 ToolEmu 风格轨迹：

```powershell
python -m agent_risk.import_public --dataset agentdojo --input data/external/agentdojo_traces.jsonl --output-dir data/public_adapted/agentdojo --manifest data/public_agentdojo_manifest.jsonl
```

## 必须做的适配性改动

1. **统一事件 schema**：不同数据源的字段名差异很大，需要全部转换为 `AgentEvent` 支持的事件类型。
2. **补充数据流标签**：公开数据通常没有资产类型、数据来源、数据去向和信任边界。
3. **补充证据步骤**：论文实验不能只给样本级标签，需要指出哪几步构成风险链路。
4. **区分强标签和弱标签**：从公开轨迹自动转换出的标签要用 `label_status` 标记，后续抽检或人工复核后才能升级。
5. **代码 Agent 场景迁移**：AgentDojo 的场景不是代码开发，需要迁移为 README、issue、依赖文件、CI、shell、网络请求等代码 Agent 场景。

## 论文实验中的使用方式

公开数据集不直接替代本项目 benchmark，而是分三种方式使用：

1. 正常行为来源：从 SWE-Hero/SWE-bench 轨迹中抽取正常代码 Agent 行为，构造 benign hard negatives。
2. 攻击模式来源：从 AgentDojo、ToolEmu、ShieldLM 中抽取攻击模板，迁移到代码 Agent 数据流场景。
3. 外部有效性验证：在公开轨迹适配后的数据上报告检测器表现，说明方法不只适用于手写样本。

## 当前风险

公开数据集的 license、字段结构和规模可能变化。正式实验前需要冻结数据版本、记录下载日期、保留转换脚本和样本哈希。对于弱标签样本，应至少进行抽样人工复核，并在论文中明确说明标签来源和局限。
